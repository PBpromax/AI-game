# AI 推理游戏开发全流程（按你现在的水平拆解）

## 一、先不要一上来就做“真 AI 大作”

你现在是大二，最稳的做法不是先追求最强模型效果，而是先把下面 4 层跑通：

1. **UI 层**：联系人列表、聊天窗口、证物栏、疑点栏、结案区
2. **接口层**：前端发消息，后端回消息
3. **规则层**：识别玩家意图、触发击穿、推进案件
4. **AI 层**：最后再把规则型回复替换成模型型回复

这样做的好处：

- 你不会被 prompt、token、模型成本拖住
- 你可以先验证玩法好不好玩
- 后面接 AI 只是“替换回复引擎”，而不是推倒重来

## 二、推荐的阶段划分

### 阶段 1：把聊天式游戏界面做出来

目标：先让玩家“像在微信里查案”这件事成立。

你需要完成：

- 左边联系人列表
- 中间聊天区
- 右边笔记区 / 调试区
- 三个置顶系统联系人：剧情 / 证物 / 疑点+结局

输出结果：

- 不接 AI 也能点联系人、切聊天、看系统信息

### 阶段 2：把后端接口打通

目标：输入一句话，后端能返回一段 NPC 回复。

你需要完成：

- `GET /api/game/bootstrap`
- `POST /api/chat/:contactId`
- `POST /api/contacts/:contactId/read`
- `POST /api/case/submit-conclusion`
- `POST /api/game/reset`

输出结果：

- 前后端真的“连起来”

### 阶段 3：实现案件推进系统

目标：让聊天不只是闲聊，而是真的能推进案件。

你需要完成：

- 玩家意图分类
- NPC 态度变化
- 关键规则命中后解锁证物
- 解锁疑点
- 解锁新联系人
- 开放结案按钮

输出结果：

- 出现完整的“提问 -> 击穿 -> 解锁 -> 再提问 -> 结案”循环

### 阶段 4：替换成真实 AI

目标：把死规则变成自然语言博弈。

你需要完成：

- 在服务端配置 Ark `responses` 接口
- 给不同 NPC 设计系统提示词
- 把 `公开信息 / 隐瞒信息 / 可击穿信息 / 口吻 / 立场 / 知识边界` 一并发给模型
- 让模型输出结构化 JSON（意图分类、回复文本、是否命中击穿）

输出结果：

- 玩家可以更自由地盘问，而不是只靠关键词触发

### 阶段 5：内容生产和打磨

目标：让它从“课程项目”升级为“可展示作品”。

你需要完成：

- 正式案件文本
- 证物配图
- UI 美术
- 音效
- 多结局
- 存档
- 发布打包

## 三、你当前最应该学会的技术点

### 前端必须会

- React 组件拆分
- `useState` / `useEffect`
- `fetch` 调接口
- 列表渲染
- 表单控制
- 条件渲染

### 后端必须会

- Express 路由
- 请求参数 / 响应 JSON
- TypeScript 类型定义
- 把“游戏状态”存在服务端
- 按规则修改状态

### 玩法设计必须会

- 把一个案件拆成：真相、公开信息、隐瞒信息、击穿条件
- 区分玩家提问是不是有效盘问
- 给玩家持续反馈，而不是一句“你问错了”

## 四、我给你这套骨架的目的

不是替你做完成品，而是帮你把最难的第一步跨过去：

- 文件结构先搭好
- 接口先打通
- 代码注释先写好
- 留空的地方也帮你标记好

你接下来最重要的任务，就是把 `mockCase.ts` 里的示例案件逐步换成你自己的设计。
