# 接口说明

## 1. 获取初始游戏状态

### `GET /api/game/bootstrap`

返回：

- 案件标题与简介
- 联系人列表
- 每个联系人的消息记录
- 已解锁证物
- 已解锁疑点
- 是否允许结案

## 2. 发送聊天消息

### `POST /api/chat/:contactId`

请求体：

```json
{
  "text": "你昨晚 22:10 为什么又回实验室？"
}
```

返回：

- 更新后的完整快照 `snapshot`
- 本次识别结果 `analysis`

## 3. 清除联系人未读

### `POST /api/contacts/:contactId/read`

作用：

- 点击联系人时把小红点清掉

## 4. 提交结案文本

### `POST /api/case/submit-conclusion`

请求体：

```json
{
  "text": "我认为陈思远利用自己保管的备用门禁再次进入实验室，出于奖金和署名分配不满拿走了原型机。"
}
```

返回：

- `status`：`pass` / `follow_up` / `fail`
- `feedback`：结案反馈
- `missingPoints`：缺失要点
- `snapshot`：最新状态

## 5. 重置案件

### `POST /api/game/reset`

作用：

- 方便开发调试
